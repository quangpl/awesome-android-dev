# Contributing to Awesome Android Development

Thank you for your interest in contributing! This list follows the [awesome manifesto](https://github.com/sindresorhus/awesome/blob/main/awesome.md). Read it before submitting a pull request.

## Guidelines

### Adding an entry

1. Submit one entry per pull request.
2. Check that the project is not already in the list.
3. Make sure your addition is genuinely awesome. "Awesome" means: actively maintained, widely adopted or uniquely valuable, and well-documented.
4. Add the entry in alphabetical order within its category (unless a logical ordering is more useful).
5. Use this exact format:

   ```
   - [Name](https://canonical-url) - One concise sentence ending with a period.
   ```

6. Link to the canonical source: the official GitHub repo or official documentation, not a tutorial, article, or mirror.
7. The description must be concise (one sentence), specific, and end with a period. Do not start with "A" or "The" when avoidable.
8. No referral links, affiliate links, or self-promotional entries without prior discussion.

### Removing or updating an entry

- Open an issue first if a project appears abandoned (no commits in 2+ years), deprecated, or replaced by a better alternative.
- Include evidence: link to the deprecation notice, successor project, or last commit date.

### New categories

- Open an issue to propose a new category before submitting a PR.
- A category needs at least 4 strong entries to be worth splitting out.

### Pull request checklist

- [ ] Entry follows the `- [Name](URL) - Description.` format.
- [ ] URL is canonical (official GitHub repo or official docs), reachable, and not a redirect chain.
- [ ] Description is one sentence, ends with a period, no em dash characters.
- [ ] Entry is placed in the correct category and in alphabetical order.
- [ ] PR title format: `Add [Name] to [Category]`
- [ ] You have verified the project is actively maintained or historically significant.

## Quality bar

Prefer entries that satisfy at least one of:
- High GitHub star count (1k+ stars as a general signal, not a hard rule).
- Official Google or JetBrains project.
- Widely cited in Android developer communities (Android Weekly, Android Developers Blog, etc.).
- Uniquely fills a gap in the ecosystem with no close substitute.

## Code of conduct

Be respectful and constructive. Entries are evaluated on technical merit, not personal preference.
